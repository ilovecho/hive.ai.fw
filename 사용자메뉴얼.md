# Hive.AI.FW 사용자 매뉴얼

IoT 디바이스 등록 → 승인 → 데이터 수집 → 실시간 모니터링 → Claude AI 분석까지,
전체 흐름을 따라 하며 익히는 사용자 안내서.

> 아키텍처·요구사항 문서는 [ANALYSIS.md](ANALYSIS.md), [REQUIREMENTS.md](REQUIREMENTS.md) 참고.

---

## 1. 전체 구성

```
[가상 IoT 디바이스]                [Hive.AI.FW 서버]                    [관리자 브라우저]
 d01_virtual_device.py            p00_aiservice.py (API·화면)          /monitor 모니터링 화면
   온습도 측정(시뮬레이션)   ──▶    ├ 디바이스 등록/승인/제한 관리        /agent   에이전트 관리 화면
   push_sensor 로 전송             ├ 수집 정보 저장 (SQLite)
                                   └ p03_ai_agent.py (Claude 분석 데몬)
                                        └─▶ Claude API (분석 요청/응답 수신)
```

| 프로그램 | 역할 | 실행 |
|----------|------|------|
| `p00_aiservice.py` | API 서버 + 관리 화면 서빙 | `python p00_aiservice.py` |
| `d01_virtual_device.py` | 가상 IoT 디바이스 (온습도 측정·전송) | `python d01_virtual_device.py` |
| `p03_ai_agent.py` | Claude 분석 에이전트 데몬 | `python p03_ai_agent.py` |
| `p02_collector.py` | (선택) 서버측 폴링 수집기 | `python p02_collector.py` |

## 2. 준비

### 2.1 필수
- Python 3.9 이상 (코어는 외부 패키지 불필요)

### 2.2 Claude AI 분석을 쓰려면 (선택)
```bash
pip install anthropic
```
그리고 API 키를 환경변수로 설정:
```bash
# Windows (PowerShell)
$env:ANTHROPIC_API_KEY = "sk-ant-..."
# Linux / Raspberry Pi
export ANTHROPIC_API_KEY="sk-ant-..."
```

### 2.3 서버 토큰 (권장)
관리 기능(승인/차단/에이전트 제어)은 토큰으로 보호된다.
```bash
# Windows (PowerShell)
$env:HIVE_AI_TOKEN = "원하는-비밀토큰"
# Linux
export HIVE_AI_TOKEN="원하는-비밀토큰"
```
토큰을 설정하지 않으면(개발용) 모든 기능이 무인증으로 열린다 — loopback 전용으로만 사용할 것.

## 3. 빠른 시작 — 10분 따라하기

터미널 3개를 연다. (2.3의 토큰은 서버 터미널에만 설정하면 된다)

### ① 서버 시작
```bash
python p00_aiservice.py
# → Hive.AI.FW 서비스 시작: http://127.0.0.1:8600/
```

### ② 가상 디바이스 시작
```bash
python d01_virtual_device.py
```
출력 예:
```
[vdev01] 등록: {'device_id': 'vdev01', 'state': 0, 'state_label': '승인대기'}
[vdev01] 전송 거부: 승인 대기 중인 디바이스입니다. 관리자 승인 후 전송할 수 있습니다.
```
**정상이다.** 디바이스는 등록만 된 상태이고, 관리자가 승인해야 데이터가 수신된다.

### ③ 브라우저에서 승인
1. `http://127.0.0.1:8600/monitor` 접속
2. 우측 상단 **[토큰 설정]** 클릭 → 서버에 설정한 `HIVE_AI_TOKEN` 값 입력
3. 디바이스 목록에서 `vdev01` 의 **[승인]** 클릭

몇 초 안에 가상 디바이스 터미널이 바뀐다:
```
[vdev01] 전송 OK  temp=26.31℃ humi=58.20%
```
모니터링 화면의 **최신 측정값** 카드와 **실시간 추이** 차트가 3초마다 갱신된다.

### ④ (선택) Claude 분석 실행
```bash
# 터미널 3 (ANTHROPIC_API_KEY 설정 필요)
python p03_ai_agent.py
```
1. `http://127.0.0.1:8600/agent` 접속 (같은 토큰)
2. **[에이전트 활성화]** 클릭 → 설정된 주기(기본 5분)마다 자동 분석
3. 또는 **[지금 분석 실행]** 클릭 → 즉시 1회 분석하고 결과를 화면에 표시

## 4. 디바이스 관리

### 4.1 디바이스 수명주기

```
register_device (디바이스 자가 등록)
      │
      ▼
 [0 승인대기] ──관리자 승인──▶ [1 승인] ──차단──▶ [9 차단]
      │                          ▲                  │
      └── push 거부              └────── 재승인 ────┘
```

- **등록**: 디바이스가 기동 시 `device_id` + `device_key`(디바이스가 만든 비밀키)로 자가 등록.
  재부팅 후 재등록해도 키가 같으면 기존 상태가 유지된다.
- **승인**: 모니터링 화면의 [승인] 버튼. 승인 전 push 는 전부 거부된다.
- **차단**: [차단] 버튼. 즉시 데이터 수신이 중단된다. [승인]으로 되살릴 수 있다.
- **제한(전송주기)**: [주기설정] 버튼으로 디바이스별 최소 전송 간격(초)을 지정.
  예: 60 으로 설정하면 60초에 1건만 받고 그보다 빠른 전송은 거부한다. 0 = 무제한.

### 4.2 실제 디바이스를 붙이려면

디바이스에서 HTTP POST 2번이면 된다 (언어 무관):

```bash
# 1) 최초 1회 등록
curl -d 'func=register_device&device_id=room1&device_key=비밀키&name=거실센서&model=DHT22' \
     http://서버:8600/

# 2) 측정할 때마다 전송 (승인 후)
curl -d 'func=push_sensor&device_id=room1&device_key=비밀키&data={"temperature":25.3,"humidity":60.1}' \
     http://서버:8600/
```

응답이 `{"status":"error","data":"승인 대기 중..."}` 이면 관리자 승인을 기다리면 된다.
`data` JSON 에 지표를 몇 개든 넣을 수 있다 (예: `{"temperature":25.3,"co2":420}`).

### 4.3 CLI 로 관리하기 (화면 없이)

```bash
TOKEN='X-HIVE-TOKEN: 비밀토큰'
curl -H "$TOKEN" -d 'func=list_device'                        http://127.0.0.1:8600/
curl -H "$TOKEN" -d 'func=approve_device&device_id=room1'     http://127.0.0.1:8600/
curl -H "$TOKEN" -d 'func=block_device&device_id=room1'       http://127.0.0.1:8600/
curl -H "$TOKEN" -d 'func=config_device&device_id=room1&push_interval=60' http://127.0.0.1:8600/
```

## 5. 모니터링 화면 (`/monitor`)

| 영역 | 내용 | 갱신 주기 |
|------|------|-----------|
| 디바이스 관리 | 상태(승인대기/승인/차단), 마지막 수신, 승인·차단·주기설정 버튼 | 10초 |
| 최신 측정값 | 디바이스·지표별 마지막 값 카드 | 3초 |
| 실시간 추이 | 선택한 디바이스·지표의 최근 60건 라인 차트 | 3초 |

- 토큰은 브라우저 탭(sessionStorage)에만 저장되며 탭을 닫으면 사라진다.
- 401 오류가 표시되면 [토큰 설정]에서 토큰을 다시 입력한다.

## 6. Claude 분석 에이전트

### 6.1 동작 방식

```
p03_ai_agent.py (데몬, 10초마다 깨어남)
  ├ heartbeat 기록  → 관리 화면에서 데몬 생존 확인
  └ 활성 상태 + 주기 도래?
      ├ 최근 window_min 분의 수집 데이터를 통계 요약
      ├ Claude API (messages) 호출 → 분석 응답 수신
      └ agent_result 테이블에 저장 → 관리 화면에 표시
```

활성/비활성 플래그는 DB(agent_config)에 있으므로 **데몬 재시작 없이**
관리 화면 버튼으로 즉시 제어된다.

### 6.2 관리 화면 (`/agent`)

| 기능 | 설명 |
|------|------|
| 활성화 / 비활성화 | 자동 분석 on/off. 비활성이어도 데몬은 대기 상태로 상주 |
| 지금 분석 실행 | 데몬과 무관하게 즉시 1회 분석 (데몬 미기동 상태에서도 동작) |
| 실행 주기(초) | 자동 분석 간격. 최소 30초 (기본 300초) |
| 분석 구간(분) | Claude 에 보낼 데이터 범위 (기본 60분) |
| Claude 모델 | 기본 `claude-opus-4-8`. 비용 절감 시 `claude-haiku-4-5` 선택 가능 |
| 분석 결과 | 최근 10건. 실행 시각·모델·토큰 사용량과 분석 본문 |

### 6.3 분석 결과 예시

> **요약**: 최근 60분간 vdev01 에서 온도 720건, 습도 720건이 정상 수집되었습니다. …
> **지표별 상태**: 온도 평균 24.8℃(22.1~27.9), 습도 평균 56.2%…
> **이상 징후**: 14:32 습도가 3분간 12%p 급락한 구간이 있습니다…
> **권장 조치**: 해당 시간대 환기/도어 이벤트 확인을 권장합니다.

### 6.4 비용 관련
- 호출 1회당 입력 약 1~2천 토큰, 출력 최대 2,048 토큰으로 제한되어 있다.
- 상시 자동 분석은 주기를 넉넉히(예: 600초 이상) 잡고, 모델을 haiku 로 낮추면 비용이 크게 준다.

## 7. func 레퍼런스 (추가분)

| func | 공개 | 파라미터 | 설명 |
|------|:--:|----------|------|
| `register_device` | 공개 | device_id, device_key, name, model | 디바이스 자가 등록 (state=승인대기) |
| `push_sensor` | 공개* | device_id, device_key, data(JSON) | 측정값 전송. *device_key + 승인 + 주기제한 검증 |
| `list_device` | 보호 | state(선택) | 디바이스 목록 |
| `approve_device` / `block_device` | 보호 | device_id | 승인 / 차단 |
| `config_device` | 보호 | device_id, push_interval | 전송 주기 제한(초) |
| `agent_status` | 보호 | - | 에이전트 상태·설정·heartbeat |
| `agent_enable` / `agent_disable` | 보호 | - | 자동 분석 on/off |
| `agent_config` | 보호 | interval_sec, window_min, model | 에이전트 설정 |
| `run_agent_now` | 보호 | - | 즉시 1회 분석 |
| `list_agent_result` | 보호 | limit | 분석 결과 목록 |

기존 func(`list_sensor`, `latest_sensor`, `infer_anomaly` 등)은 [README.md](README.md) 참고.
보호 func 는 `X-HIVE-TOKEN` 헤더 필요 (서버에 `HIVE_AI_TOKEN` 설정 시).

## 8. 운영 (Raspberry Pi / systemd)

```bash
sudo cp /opt/hive.ai.fw/etc/hive-ai-*.service /etc/systemd/system/
# hive-ai-agent.service 의 ANTHROPIC_API_KEY, 각 유닛의 HIVE_AI_TOKEN 수정 후:
sudo systemctl daemon-reload
sudo systemctl enable --now hive-ai-api hive-ai-collector hive-ai-agent
```

가상 디바이스는 데모용이므로 서비스 등록 없이 필요할 때만 실행한다.

## 9. 트러블슈팅

| 증상 | 원인 / 조치 |
|------|-------------|
| 디바이스가 계속 "전송 거부: 승인 대기" | 정상 동작. `/monitor` 에서 [승인] 클릭 |
| "전송 주기 제한: n초 이후..." | [주기설정] 값보다 빠르게 전송 중. 디바이스 주기를 늘리거나 제한을 0 으로 |
| 화면에서 401 "인증이 필요합니다" | [토큰 설정]에 서버의 `HIVE_AI_TOKEN` 과 같은 값 입력 |
| "anthropic 패키지가 없습니다" | `pip install anthropic` |
| "ANTHROPIC_API_KEY 환경변수가..." | 데몬/서버를 띄운 터미널에 API 키 설정 후 재시작 |
| "최근 n분간 수집된 데이터가 없습니다" | 가상 디바이스가 승인·전송 중인지 확인 후 다시 분석 |
| heartbeat "데몬 미기동" | `python p03_ai_agent.py` 실행. [지금 분석 실행]은 데몬 없이도 동작 |
| Claude API 사용량 제한(429) | 잠시 후 재시도. 자동 분석 주기를 늘릴 것 |
| 화면이 안 열림 (`/monitor` 404) | `w01_monitor.html` 이 서버와 같은 폴더에 있는지 확인 |
